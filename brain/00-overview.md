# 00 — نظرة عامة على النظام

## بجملة واحدة
نظام ويب مؤسسي لوزارة واحدة، يدير عدّة أقسام تنظيمية، كل قسم يرى بياناته فقط (عزل صارم)، قابل للتوسعة بموديولات تُفعَّل وتُضبَط لكل قسم بلا إعادة كتابة.

## لمن؟
- **موظفو الأقسام:** يعملون ضمن نطاق قسمهم (Department-Scoped).
- **مديرو الأقسام:** يديرون موارد قسمهم وصلاحياته.
- **مدير النظام (Super Admin):** يضبط الموديولات والأقسام والصلاحيات على مستوى الوزارة.

## القرارات الحاكمة (محسومة — لا تُعاد)
1. **وزارة واحدة، لا Multi-tenancy.** لكن **Row-Level Security على مستوى القسم** صارم: لا سجل بلا `OwnerUnitId`، والتصفية تلقائية على الخادم عبر EF Global Query Filter.
2. **Vertical Slice أولًا.** نبني موديولًا كاملًا end-to-end يثبت المعمارية ثم نُعمّمه كـ Template.
3. **Primitives قبل Features.** DataGrid · Drawer · Modal · FormBuilder · ReportEngine تُبنى مرة وتُستهلَك.
4. **Boring Technology + Measure don't guess.** كل ادّعاء أداء يُثبَت برقم تحت حمل.
5. **Opt-in لا Mandatory.** الميزات الثقيلة تُفعَّل لكل شاشة عند الحاجة.

## أهداف الأداء (SLOs)
| الفئة | الهدف (p95) |
|---|---|
| Read بسيط | < 200ms |
| Read مُركّب | < 400ms |
| Write | < 300ms |
| تقرير/Export | Async عبر Job (لا SLA متزامن) |
