# ADR-0003 — تضمين نطاق العزل (Unit Scope) في الـ JWT

- **الحالة:** مقبول
- **التاريخ:** Phase 1

## السياق
الـ Global Query Filter للعزل (RLS) يحتاج نطاق المستخدم (قائمة الوحدات المسموح برؤيتها) **أثناء كل استعلام**. مصدر النطاق هو شجرة Org Units. لكن `ICurrentUser` يُستدعى **داخل الفلتر نفسه**، فلا يمكنه استعلام قاعدة البيانات (إعادة دخول/حلقة).

## القرار
يُحسب النطاق **مرة واحدة عند تسجيل الدخول** (شجرة وحدة المستخدم الفرعية عبر المسار المادي `Path`) ويُضمَّن في الـ access token كـ claims (`unit_scope`). `CurrentUser` يقرأه من الـ claims بلا أي استعلام.

## البدائل المرفوضة
- **استعلام النطاق لكل طلب من DB:** يتعارض مع كون `ICurrentUser` يُستخدم داخل الفلتر؛ ويضيف round-trip لكل طلب.
- **تخزين النطاق في cache/session على الخادم:** حالة خادمية تكسر انعدام الحالة (stateless) وتُعقّد التوسّع الأفقي.

## العواقب
- النطاق ثابت طوال عمر الـ access token (دقائق). تغيّر هيكل وحدة المستخدم يسري بعد التحديث التالي للرمز (access قصير العمر ⇒ تأخير محدود).
- حجم الرمز ينمو مع عدد وحدات النطاق — مقبول لهيكل وزارة واحدة. لو ضخم مستقبلًا: نضغط النطاق إلى مدى/جذر بدل قائمة.
- مسارات المصادقة (login/refresh/compute-scope) تتجاوز فلتر العزل صراحةً (`IgnoreQueryFilters`) — عمليات نظام مُبرَّرة ومُوثَّقة.
