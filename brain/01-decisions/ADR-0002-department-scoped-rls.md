# ADR-0002 — العزل: Department-Scoped RLS (لا Multi-tenancy)

- **الحالة:** مقبول (محسوم)
- **التاريخ:** Phase 0

## السياق
وزارة واحدة بعدّة أقسام لا يجب أن ترى بيانات بعضها (قسم التعليم الإلكتروني ≠ قسم تسجيل الطلاب)، مع تسلسل هرمي إداري.

## القرار
عزل على مستوى الصف (Row-Level Security) محدود بالوحدة التنظيمية:
- جدول **Org Units** هرمي.
- كل سجل قابل للعزل يحمل **`OwnerUnitId`** إلزاميًّا.
- **EF Core Global Query Filter** يُصفّي تلقائيًا حسب نطاق المستخدم.
- التحقّق على الخادم حصرًا.

تفاصيل التطبيق في `brain/05-rls-and-permissions.md`.

## البدائل المرفوضة
- **Multi-tenancy (DB/schema لكل قسم):** مبالغة — كيان واحد (وزارة)، يضاعف التشغيل بلا حاجة.
- **تصفية يدوية في كل استعلام:** هشّ، يُنسى، يُسرّب البيانات. Global Filter يجعل العزل افتراضيًا آمنًا.
- **تصفية في الواجهة:** ثغرة أمنية مباشرة.

## العواقب
- أمان افتراضي (fail-closed): نسيان التصفية لا يُسرّب لأن الفلتر عام.
- أي تجاوز يتطلب `IgnoreQueryFilters` صريحًا ومُدقَّقًا.
- يجب حقن سياق المستخدم (`ICurrentUser`) في `DbContext`.
